Módulo 1 · Introdução

O que é o n8n?

Aprenda automação visual do zero: conceitos, instalação, criação de workflows e deploy em produção.

n8n (pronuncia-se "nodemation") é uma ferramenta de automação de fluxos open-source. Pense nele como um Lego para conectar serviços — você liga APIs, bancos de dados e aplicativos entre si sem precisar escrever código na maioria dos casos.

🔓

Open Source

Código aberto. Roda no seu servidor. Seus dados ficam com você.

🔌

400+ Integrações

Telegram, Google Sheets, Slack, HTTP, PostgreSQL, AWS e muito mais.

💻

Low-code

Interface visual com drag-and-drop. Código JS opcional para lógica avançada.

🐳

Fácil de instalar

Um único docker compose up e está no ar em minutos.

n8n vs. outros

Módulo 2 · Fundamentos

Conceitos Principais

Entenda o vocabulário do n8n antes de começar a construir.

🔲 Node (Nó)

A unidade básica do n8n. Cada node faz uma coisa: disparar um evento, fazer uma requisição HTTP, transformar dados, enviar mensagem, etc.

⚡ Trigger
Inicia o fluxo
→
🔧 Action
Faz algo
→
🔀 IF
Decide
→
📤 Output
Envia resultado

📋 Tipos de Nodes

TipoO que fazExemplos
Trigger Inicia o workflow quando algo acontece Webhook, Cron, Telegram Trigger, Email recebido
Action Executa uma ação em um serviço Enviar email, criar linha no Google Sheets, POST HTTP
Transform Transforma ou filtra os dados Code, Set, Filter, Merge, Split
Flow Control Controla o fluxo lógico IF, Switch, Loop, Wait, SubWorkflow

🔀 Workflow

Um workflow é uma coleção de nodes conectados que definem um processo automatizado. Pode ser ativado manualmente, por agendamento (cron) ou por evento externo (webhook).

✅
Exemplo real: Toda vez que alguém envia uma mensagem no Telegram → o n8n recebe via Trigger → verifica se é um vídeo → baixa o arquivo → processa → responde automaticamente.

📦 Item (dado)

Os dados fluem entre nodes como Items — objetos JSON. Cada node recebe um array de items e pode transformá-los, filtrar, ou adicionar campos.

Exemplo de Item JSON
JSON 
{
  "json": {
    "message": {
      "chat_id": 123456789,
      "text": "Olá!",
      "from": { "username": "joao" }
    }
  },
  "binary": {}  // arquivos ficam aqui
}

🔗 Expressions (Expressões)

Para referenciar dados de nodes anteriores, você usa expressões entre duplas chaves:

Sintaxe de expressão
// Pega valor do node anterior
{{ $json.message.chat_id }}

// Pega valor de um node específico
{{ $node["Telegram Trigger"].json.message.text }}

// JavaScript inline
{{ $json.nome.toUpperCase() }}
Módulo 3 · Infraestrutura

Docker — Fundamentos

O básico que você precisa saber para rodar o n8n.

Docker é uma plataforma que empacota aplicações em containers — ambientes isolados que contêm tudo que o app precisa para rodar, independente do sistema operacional do host.

📦

Image (Imagem)

Receita somente leitura. Define o sistema base + app + dependências.

🏃

Container

Instância em execução de uma imagem. Isolado do sistema host.

💾

Volume

Armazenamento persistente. Dados sobrevivem ao reiniciar o container.

📝

Compose

Arquivo YAML que define múltiplos containers e suas configurações.

Instalando o Docker

🐧 Linux (Ubuntu/Debian)
# Instala Docker com o script oficial
curl -fsSL https://get.docker.com | sh

# Adiciona seu usuário ao grupo docker (sem precisar de sudo)
sudo usermod -aG docker $USER
newgrp docker

# Verifica instalação
docker --version
docker compose version
🍎 macOS / 🪟 Windows
# Baixe e instale o Docker Desktop em:
https://www.docker.com/products/docker-desktop/

# Inclui Docker Engine + Docker Compose automaticamente

Comandos essenciais

ComandoO que faz
docker compose up -dSobe todos os serviços em background
docker compose downPara e remove os containers
docker compose logs -fVê os logs em tempo real
docker compose restartReinicia os serviços
docker compose psLista containers e status
docker compose exec n8n shAbre terminal dentro do container
docker system pruneLimpa containers/imagens não usados
💡
Dica: O -d em up -d significa "detached" — roda em background sem travar o terminal. Sem ele, o processo prende o terminal e para quando você fechar.

Anatomia do docker-compose.yml

📄 docker-compose.yml
YAML 
version: "3.8"

services:          # Lista de containers
  meu-app:
    image: nginx   # Imagem do Docker Hub
    container_name: app_nginx
    restart: unless-stopped  # Reinicia se cair
    ports:
      - "8080:80"   # porta_host:porta_container
    environment:   # Variáveis de ambiente
      - MINHA_VAR=valor
    volumes:       # Mapeamento de pastas
      - ./dados:/app/dados  # pasta_local:pasta_container
      - volume_nomeado:/var/data

volumes:          # Volumes nomeados (gerenciados pelo Docker)
  volume_nomeado:
Módulo 4 · Instalação

Instalando o n8n

Do zero ao n8n rodando em menos de 5 minutos.

1

Crie a pasta do projeto

Terminal
mkdir meu-n8n && cd meu-n8n
2

Crie o arquivo docker-compose.yml

📄 docker-compose.yml
version: "3.8"

services:
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=0.0.0.0
      - N8N_PORT=5678
      - GENERIC_TIMEZONE=America/Sao_Paulo
      - N8N_ENCRYPTION_KEY=mude_isso_para_algo_secreto
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:
3

Suba o container

Terminal
docker compose up -d

# Acompanhe os logs (aguarde "n8n ready")
docker compose logs -f
4

Acesse e crie sua conta

Abra no navegador: http://localhost:5678

  • Preencha nome, email e senha
  • Esse será o admin do seu n8n
  • Dados salvos no volume Docker — persistem reinicializações
🎉
Pronto! Seu n8n está rodando. Agora vamos conhecer a interface.

Node: HTTP Request

O mais versátil. Faz requisições para qualquer API REST.

Configuração típica
Method: POST
URL: https://api.exemplo.com/dados
Authentication: Bearer Token
Body (JSON):
{
  "nome": "{{ $json.nome }}",
  "email": "{{ $json.email }}"
}

Node: Code (JavaScript)

Para lógica customizada que não existe em nodes prontos.

Exemplo: transformar dados
// $input.all() retorna todos os items
// Deve retornar array de items
const items = $input.all();

return items.map(item => ({
  json: {
    nome_completo: `${item.json.primeiro} ${item.json.ultimo}`,
    email: item.json.email.toLowerCase(),
    processado_em: new Date().toISOString()
  }
}));

Node: IF (Condicional)

Divide o fluxo em dois caminhos baseado em uma condição.

Configuração
Condition:
  Value 1: {{ $json.tipo }}
  Operation: equals
  Value 2: "video"

// Saída True  → node para processar vídeo
// Saída False → node de resposta de erro

Node: Schedule Trigger (Cron)

Exemplos de agendamento
# Todo dia às 9h
0 9 * * *

# A cada 30 minutos
*/30 * * * *

# Toda segunda-feira às 8h
0 8 * * 1

# Ou use o modo "Interval": a cada X horas/minutos
Módulo 7 · Prática

Criando seu Primeiro Workflow

Passo a passo: um bot que responde mensagens no Telegram.

🎯
Objetivo: Criar um workflow que recebe mensagens no Telegram e responde automaticamente com um eco do que foi enviado.
1

Crie o bot no Telegram

  • Abra o Telegram e procure @BotFather
  • Digite /newbot
  • Dê um nome e username (ex: MeuTesteBot)
  • Copie o token gerado
2

Adicione a credencial no n8n

  • Acesse: Settings → Credentials → Add
  • Escolha Telegram API
  • Cole o token do BotFather
  • Nomeie como Meu Bot Telegram
  • Clique em Save
3

Crie um novo workflow

  • Clique em + New Workflow
  • Clique em + Add first step
  • Busque por Telegram Trigger
  • Selecione a credencial criada
  • Em Updates, marque message
4

Adicione o node de resposta

  • Clique no + na saída do Telegram Trigger
  • Busque por Telegram (action, não trigger)
  • Operation: Send Message
  • Chat ID: {{ $json.message.chat.id }}
  • Text: Você disse: {{ $json.message.text }}
5

Ative e teste

  • Clique em Save
  • Ative o toggle no canto superior direito
  • Mande uma mensagem pro seu bot no Telegram
  • 🎉 Deve responder automaticamente!

Fluxo resultante

💬 Telegram Trigger
message
→
📤 Telegram
Send Message
⚠️
Importante: O Telegram Trigger usa webhook. Para funcionar localmente, o n8n precisa ser acessível pela internet. Para testes locais, use ngrok para expor a porta 5678.
Módulo 8 · Dados

Expressões & Dados

Como acessar, transformar e manipular dados entre nodes.

Variáveis disponíveis

VariávelO que acessa
$jsonDados JSON do item atual (saída do node anterior)
$binaryDados binários (arquivos, imagens) do item atual
$input.all()Todos os items de entrada como array
$input.first()Primeiro item de entrada
$node["Nome"]Acessa dados de um node específico por nome
$workflow.idID do workflow atual
$execution.idID único desta execução
$nowData/hora atual (objeto DateTime)

Exemplos práticos

Acessando dados
// Texto da mensagem do Telegram
{{ $json.message.text }}

// Chat ID (para responder)
{{ $json.message.chat.id }}

// Data formatada
{{ $now.toFormat('dd/MM/yyyy HH:mm') }}

// Acessar node específico
{{ $node["HTTP Request"].json.resultado }}

// Primeiro item de entrada
{{ $input.first().json.nome }}
Transformações JS inline
// Maiúsculas
{{ $json.nome.toUpperCase() }}

// Condicional ternário
{{ $json.ativo ? 'Ativo' : 'Inativo' }}

// Template literal
{{ `Olá, ${$json.nome}! Bem-vindo.` }}

// Acessar array
{{ $json.tags[0] }}

// Valor padrão se null
{{ $json.email ?? 'sem_email@n.com' }}

No node Code (JS completo)

Exemplo: processar lista
const items = $input.all();
const resultado = [];

for (const item of items) {
  const { nome, valor } = item.json;
  
  // Processa cada item
  if (valor > 100) {
    resultado.push({
      json: {
        nome,
        valor,
        categoria: 'alto',
        processado: new Date().toISOString()
      }
    });
  }
}

return resultado;
💡
Dica: No campo de qualquer node, clique no botão { } ao lado do campo para alternar entre modo fixo e modo expressão.
Módulo 9 · Segurança

Credenciais & Segredos

Como gerenciar tokens e senhas com segurança.

Credenciais no n8n são armazenadas criptografadas usando a N8N_ENCRYPTION_KEY que você define. Nunca aparecem em logs ou no JSON exportado.

Adicionando uma credencial

1

Vá em Settings → Credentials → Add credential

2

Busque o tipo: Telegram API, Google OAuth, HTTP Header Auth, etc.

3

Preencha os campos e clique em Save. Pronto para usar em qualquer workflow.

Tipos comuns de credencial

ServiçoTipo de authO que precisar
Telegram BotAPI TokenToken do @BotFather
Gmail / GoogleOAuth2Client ID + Secret no Google Cloud
OpenAIAPI KeyChave da plataforma.openai.com
Qualquer API RESTHeader AuthAuthorization: Bearer TOKEN
PostgreSQLDB CredentialsHost, porta, user, senha, database
⚠️
Nunca perca sua N8N_ENCRYPTION_KEY! Se perder, as credenciais salvas ficam inacessíveis e você precisará recriar todas. Guarde em local seguro.
Módulo 10 · Deploy

Indo para Produção

Como deixar seu n8n rodando 24/7 de forma confiável.

Setup completo para servidor

📄 docker-compose.yml (produção)
version: "3.8"

services:
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=0.0.0.0
      - N8N_PORT=5678
      - N8N_PROTOCOL=https
      - WEBHOOK_URL=https://seudominio.com/
      - GENERIC_TIMEZONE=America/Sao_Paulo
      - N8N_ENCRYPTION_KEY=${N8N_ENCRYPTION_KEY}
      - N8N_PAYLOAD_SIZE_MAX=2048
      - EXECUTIONS_TIMEOUT=1800
      - EXECUTIONS_DATA_SAVE_ON_SUCCESS=none  # economiza espaço
      - EXECUTIONS_DATA_MAX_AGE=72              # deleta logs antigos
    volumes:
      - n8n_data:/home/node/.n8n
      - ./scripts:/scripts:ro
      - ./tmp:/tmp_work
    env_file:
      - .env  # variáveis secretas em arquivo separado

volumes:
  n8n_data:
📄 .env (não suba para o git!)
N8N_ENCRYPTION_KEY=gere_uma_chave_aleatoria_longa_aqui

# Gere com:
# openssl rand -base64 32

Adicionando HTTPS com Nginx (recomendado)

📄 nginx.conf
server {
    listen 443 ssl;
    server_name seudominio.com;

    ssl_certificate /etc/letsencrypt/live/seudominio.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/seudominio.com/privkey.pem;

    location / {
        proxy_pass http://localhost:5678;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        client_max_body_size 2G;  # para uploads grandes
    }
}

server {
    listen 80;
    server_name seudominio.com;
    return 301 https://$host$request_uri;
}
Certificado SSL gratuito com Certbot
# Instala certbot
sudo apt install certbot python3-certbot-nginx

# Gera certificado
sudo certbot --nginx -d seudominio.com

# Renova automaticamente (já configurado pelo certbot)
sudo systemctl status certbot.timer

Backup dos dados

Script de backup
#!/bin/bash
# Exporta workflows como JSON
docker exec n8n n8n export:workflow --all --output=/home/node/.n8n/backup_workflows.json

# Faz backup do volume completo
docker run --rm \
  -v n8n_data:/data \
  -v $(pwd)/backups:/backup \
  alpine tar czf /backup/n8n_backup_$(date +%Y%m%d).tar.gz /data
