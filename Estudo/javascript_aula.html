// 01 — contexto

Por que JavaScript existe?

👨‍🏫 Professor

Imagina que você chegou num site bonito, clicou num botão e nada aconteceu. Parece morto, né? Isso é uma página só com HTML e CSS. Agora imagina que ao clicar o botão muda de cor, aparece uma mensagem, um item é adicionado numa lista... Isso é JavaScript fazendo seu trabalho.

HTML é a estrutura — os ossos. CSS é a aparência — a roupa. JavaScript é o comportamento — o movimento. Toda interação que você já viu num site (menus que abrem, formulários que validam, animações que respondem ao clique) tem JavaScript por baixo.

HTML

Estrutura

Define o que existe na página: títulos, parágrafos, botões, imagens. O esqueleto.

CSS

Aparência

Define como as coisas parecem: cores, tamanhos, posições, animações visuais.

JavaScript

Comportamento

Define o que acontece quando o usuário age: clica, digita, move o mouse.

DOM

A ponte

O DOM é a representação da página que o JS usa para acessar e modificar qualquer elemento.

// 02 — conceito fundamental

O que diabos é o DOM?

👨‍🏫 Professor

DOM significa Document Object Model. Esquece o nome por agora. O que importa é: quando o navegador lê seu HTML, ele transforma tudo numa árvore de objetos. Cada elemento vira um objeto que o JavaScript pode acessar, modificar ou remover. É como se o JavaScript tivesse um mapa de toda sua página.

Pensa assim: seu HTML tem um <h1> com o texto "Olá". O DOM é o que permite que você pegue esse <h1> usando JavaScript e mude o texto para "Oi, mundo!" sem recarregar a página.

index.html 
<!-- Esse é o seu HTML -->
<h1 id="meu-titulo">Olá</h1>
<button id="meu-botao">Clica aqui</button>

<!-- O JavaScript fica aqui embaixo -->
<script>
  // 1. Pega o elemento pelo id
  const titulo = document.getElementById("meu-titulo")

  // 2. Pega o botão
  const botao = document.getElementById("meu-botao")

  // 3. Diz o que acontece quando clicar
  botao.addEventListener("click", function() {
    titulo.textContent = "Você clicou!"
  })
</script>
🔍 Leia linha a linha: document.getElementById("meu-titulo") está dizendo — "ei, documento, me dá o elemento que tem id igual a 'meu-titulo'". Simples assim. O addEventListener("click", ...) está dizendo — "quando alguém clicar nesse botão, execute essa função".

Isso não é abstrato. Veja funcionando agora:

Percebeu? O botão não recarregou a página. O JavaScript pegou o elemento já existente no DOM e alterou suas propriedades. Isso é exatamente o que você vai aprender a fazer.

// 03 — como selecionar elementos

Três formas de achar um elemento

👨‍🏫 Professor

Antes de modificar qualquer coisa, você precisa achar o elemento. É como procurar um livro numa biblioteca — você pode procurar pelo número da prateleira, pelo título, ou pelo assunto. No DOM temos três jeitos principais.

seletores.js 
// JEITO 1: pelo ID (único — só um elemento por id)
const titulo = document.getElementById("meu-id")

// JEITO 2: pela classe (pode pegar vários)
const itens = document.querySelectorAll(".minha-classe")

// JEITO 3: querySelector (usa seletor CSS — mais flexível)
const primeiro = document.querySelector("#lista li:first-child")

Na prática, você vai usar getElementById e querySelector 90% do tempo. O querySelector é o mais poderoso porque aceita qualquer seletor CSS.

// Propriedades mais usadas para modificar elementos

.textContent Modifica o texto dentro de um elemento. "Olá" vira o que você quiser.
.innerHTML Modifica o HTML interno — permite inserir tags dentro do elemento.
.style.propriedade Modifica qualquer estilo CSS diretamente. Ex: .style.color = "red"
.classList Adiciona, remove ou alterna classes CSS. Muito usado para mostrar/esconder coisas.
// 04 — eventos: como reagir ao usuário

Eventos são a alma do JavaScript

👨‍🏫 Professor

Um evento é qualquer coisa que acontece na página: um clique, uma tecla pressionada, o mouse passando em cima de algo, o usuário digitando num input. O JavaScript fica "escutando" esses eventos e executa uma função quando eles acontecem. Por isso usamos addEventListener — você está dizendo ao navegador: "fique de olho nisso e avise quando acontecer".

eventos.js 
const input = document.querySelector("#meu-input")
const lista = document.querySelector("#minha-lista")

input.addEventListener("keydown", function(evento) {
  // O parâmetro "evento" carrega informações sobre o que aconteceu
  if (evento.key === "Enter") {
    const texto = input.value  // pega o que foi digitado

    if (texto !== "") {
      const item = document.createElement("li")  // cria elemento novo
      item.textContent = texto
      lista.appendChild(item)  // adiciona na lista
      input.value = ""  // limpa o campo
    }
  }
})
💡 Preste atenção no document.createElement("li") — isso cria um novo elemento HTML do nada, via JavaScript. Depois appendChild coloca ele dentro de outro elemento. Isso é a base de como frameworks modernos como React funcionam por baixo dos panos.

// 05 — hora de construir

Sua atividade prática

👨‍🏫 Professor

Tudo que você aprendeu até aqui — selecionar elementos, modificar propriedades e reagir a eventos — é suficiente para construir algo real. A tarefa é criar um Mini App de Lista de Habilidades. Parece simples? É. E é exatamente por isso que é perfeito para fixar o que você aprendeu.

// tarefa

Mini App: Lista de Habilidades

Crie um único arquivo index.html com HTML, CSS e JavaScript que faça o seguinte:

  • 1
    Estrutura HTML: crie um título, um campo de texto (input), um botão "Adicionar" e uma lista vazia (ul) onde os itens vão aparecer.
  • 2
    Estilo CSS: dê um visual agradável — não precisa ser elaborado, mas mostre que você pensou na apresentação. Centralizar na tela já vai bem.
  • 3
    Adicionar item: quando o usuário clicar no botão (ou pressionar Enter), o texto do input deve aparecer como um novo item na lista.
  • 4
    Remover item: cada item da lista deve ter um botão "×" ao lado. Ao clicar, o item deve ser removido da lista.
  • 5
    Validação simples: não permita adicionar itens vazios. Se o campo estiver em branco, não faça nada (ou mostre uma mensagem).
    // dica do professor: Use if (input.value.trim() !== "") para checar se tem texto. O .trim() remove espaços em branco do início e fim.
  • ★
    Desafio bônus: adicione um contador que mostra quantos itens estão na lista. Atualize ele toda vez que adicionar ou remover.
    // dica do professor: Use lista.children.length para saber quantos filhos um elemento tem. Atualize um elemento de texto com esse valor após cada ação.
🎯

O objetivo não é ter um resultado perfeito — é perceber que você consegue fazer uma página fazer algo. Cada bug que você resolve, cada comportamento que funciona como você imaginou, é você aprendendo a pensar como programador.
